Mysql数据库基础了解
Mysql
数据库
Publish Date:   2019-10-13

数据库基础语法

数据库（Database）是按照数据结构来组织、存储和管理数据的仓库。每个数据库都有一个或多个不同的 API 用于创建，访问，管理，搜索和复制所保存的数据。

数据库存储引擎

插件式的存储引擎,存储引擎可以基于一个表、一个库,同一表的不同地区。
InnoDB存储引擎 (默认引擎)
支持外键;
支持事务,如果需要执行大量的增删改操作,出于事务的安全考虑,建议使用InnoDB存储引擎;
MyISAM
具有检查和修复的大多数工具;
表可以被压缩;
支持全文索引;
但是不支持外键和事务;
主要是用于执行大量的查询操作;
set default_storage_engine=MyISAM,设置当前会话引擎为MYISAM;

SQL基础语句

SQL (Structured Query Language)的定义

结构化查询语句

  1. 数据定义语句DDL
    create、alter、drop、truncate
  2. 数据操作语句
    insert、delete、update、select
  3. 数据控制语句
    grant(授权)、revoke(收回权限)
  4. 事务控制语句
    begin(开始事务)、commit(提交)、rollback(回滚)

    相关语句

1.库操作
create datebase 数据库名+默认约束(字符类型)default charset=’utf8’;
show create datebase 数据库名; 查看建的数据库属性
drop datebase 数据库名; 删除数据库

2.mysql数据类型:
数值类型、字符串类型、日期类型、复合类型(enum类型和set类型)、二进制类型
decimal(length,precision)表示数据精度 (5,2)去5位包括2位小数点
char() 固定长度
varchar()可变长度

3.表操作
1.先选择所用库
use 数据库名;
2.建表
create table 表名(
字段名1 字段约束,
字段名2 字段约束
)表约束(存储引擎、字符集等);
3.查看表结构
desc 表名;show create table 表名;
4.删除表以及字段等操作
删除表: drop table 表名;
删除字段: alter table 表名 drop 字段名;
新增字段: alter table 表名 add 新字段名 字段约束;
修改字段名: alter table 表名 change 旧字段名 新字段名 新字段约束;
修改字段类型: alter table 表名 modify 字段名 新字段约束;
修改表名: rename table 表名 to新表名=alter table 表名 rename 新表名;
新增数据: insert into 表名 字段列表 values (值列表);
从源表导入数据: insert into 表名 字段列表 select 字段列表 from 表名 where 条件表达式;
复制表结构和数据: insert into 表名 select * from 表名 (where 1= 2);
修改表数据(记录): update 表名 set 字段1=值,字段2=值 [where 条件表达式];
删除表记录: delete from 表名 [where 条件表达式] (只删除表记录、可进行回滚);
删除表记录: truncate table 表名(无法进行回滚,不能接条件表达式.重置自增长);

1.新建库
2.use 新库
3.source 文件地址
